# Per-Shot Generation + ffmpeg Stitching Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the one-Seedance-call-with-all-shots-crammed-in scene route (shipped in `docs/superpowers/plans/2026-06-25-anti-slop-multishot-remediation.md`, Tasks 6-7) with the real film/TV editing model: generate each shot as its own full-length clip (still carrying `reference_images` for character consistency), then stitch the resulting clips into one continuous file via `ffmpeg` concatenation.

**Architecture:** The scene POST route submits one `generateTextVideo` call per shot (each shot's own `videoPrompt`, own `duration`, shared `referenceImages` — no `multiShotPrompt`), writing each shot's job onto its own row exactly like the pre-existing per-shot route already does. The scene status GET route polls every shot's own job, and once every shot in the scene has a `finalVideoUrl`, downloads all of them, concatenates with `ffmpeg-static` (stream-copy, no re-encoding — all clips share the same Seedance-produced codec/resolution), uploads the result to Vercel Blob, and writes it to a new `sceneFinalVideoUrl` field shared across the scene's shots.

**Tech Stack:** `ffmpeg-static` (npm, bundles a static binary) + Next.js `outputFileTracingIncludes` (the documented Vercel/Next.js pattern for bundling a native binary into a serverless function — works around a known path-resolution bug when relying on default tracing), `child_process.spawn`.

## Global Constraints

- Found live during the prior plan's real-money A/B test: 1 Seedance call with all 6 shots crammed into a single ≤15s clip produces ~2.5s/shot — too compressed for a deliberately-paced cinematic trailer (works fine for Segmind's own stated "Fast · Social content · Best for shorts" use case, but not this one). User explicitly chose the per-shot-then-stitch redesign over further tuning the one-call approach.
- Verified via Vercel/Next.js official reference (`vercel-labs/ffmpeg-on-vercel`) and a documented GitHub issue (`vercel/next.js#53791`, statically-bundled binaries failing path resolution in serverless builds): `ffmpeg-static`'s binary path must be declared in `next.config.js`'s `outputFileTracingIncludes`, keyed by the exact route path, or the bundled binary will not be found at runtime on Vercel even though it works locally.
- Concatenation must use `-c copy` (stream copy / remux, no re-encoding) — all inputs are Seedance-2.0-produced MP4s at the same settings, so re-encoding would be wasted compute and cost. Do not add a transcode step.
- `multiShotScript` (the prior plan's Task 1-3 column/prompt feature) is **not removed** in this plan — it's no longer consumed by the new per-shot route (replaced by each shot's own `videoPrompt`), but ripping it out is unrelated cleanup, not in scope here. It is harmless and inert (still generated and stored, just unread by the new code path).
- Existing per-shot `generate-video`/`generate-video/status` routes (used by every non-seedance model, and by seedance via the per-shot "Generate Video" button) remain **completely unchanged** — this plan only touches the scene-level routes.
- `tsc --noEmit` must exit 0 and the existing test suite must stay green throughout.
- `referenceImages` (deduped character portrait URLs, capped at 9) and the "only `character.portraitUrl`, never a user photo" guarantee from the prior plan carry over unchanged.

```ts
import { spawn } from "node:child_process";
import { writeFile, unlink } from "node:fs/promises";
import ffmpegPath from "ffmpeg-static";

/**
 * Concatenates videos via ffmpeg's concat demuxer with stream-copy (-c copy,
 * no re-encoding) — only valid when every input shares the same codec/
 * resolution/timebase, true here since all inputs are Seedance-2.0 outputs
 * generated under this app's own fixed request shape.
 */
export async function concatVideos(inputPaths: string[], outputPath: string): Promise<void> {
  const listPath = `${outputPath}.list.txt`;
  const listContent = inputPaths.map(p => `file '${p}'`).join("\n");
  await writeFile(listPath, listContent);

  await new Promise<void>((resolve, reject) => {
    const proc = spawn(ffmpegPath as unknown as string, [
      "-y",
      "-f", "concat",
      "-safe", "0",
      "-i", listPath,
      "-c", "copy",
      outputPath,
    ]);

    let stderr = "";
    proc.stderr.on("data", (chunk: Buffer) => { stderr += chunk.toString(); });
    proc.on("error", reject);
    proc.on("close", (code: number) => {
      if (code === 0) resolve();
      else reject(new Error(`ffmpeg concat failed (exit ${code}): ${stderr}`));
    });
  });

  await unlink(listPath);
}
```

```ts
export const dynamic = 'force-dynamic';
export const maxDuration = 300;

import { NextResponse } from "next/server";
import { getRequiredSession } from "@/lib/auth-helpers";
import { db } from "@/db";
import { projects, productionShots, users } from "@/db/schema";
import { eq, and } from "drizzle-orm";
import { generateTextVideo } from "@/lib/higgsfield/client";
import { decrypt } from "@/lib/crypto";

async function verifyOwnership(projectId: string, userId: string) {
  return db.query.projects.findFirst({
    where: and(eq(projects.id, projectId), eq(projects.userId, userId)),
  });
}

// Each shot in the scene gets its OWN full-length Seedance 2.0 call (with
// reference_images for character consistency) instead of being crammed into
// one shared multi-shot prompt — confirmed live (see
// docs/superpowers/plans/2026-06-25-anti-slop-multishot-remediation.md) that
// 6 shots in one <=15s clip reads as too compressed for a deliberately-paced
// trailer. The status route (Task 5 of the per-shot-stitching plan) stitches
// the resulting independent clips together once all are ready.
export async function POST(_: Request, { params }: { params: Promise<{ projectId: string; sceneNumber: string }> }) {
  const s = await getRequiredSession();
  const { projectId, sceneNumber: sceneNumberRaw } = await params;
  if (!await verifyOwnership(projectId, s.user.id))
    return NextResponse.json({ error: "Not found" }, { status: 404 });

  const sceneNumber = Number(sceneNumberRaw);

  const sceneShots = await db.query.productionShots.findMany({
    where: and(eq(productionShots.projectId, projectId), eq(productionShots.sceneNumber, sceneNumber)),
    with: { primaryCharacter: true },
    orderBy: (sh, { asc }) => [asc(sh.shotNumber)],
  });
  if (sceneShots.length === 0) return NextResponse.json({ error: "Scene not found" }, { status: 404 });

  const user = await db.query.users.findFirst({ where: eq(users.id, s.user.id) });
  const segmindKey = decrypt(user?.segmindApiKey ?? "");
  if (!segmindKey)
    return NextResponse.json({ error: "Add your Segmind API key in Settings." }, { status: 400 });

  // Only character.portraitUrl (AI-generated via generateSoulImage, this feature's
  // own portrait pipeline) is ever used as a reference — never a user-supplied photo.
  const referenceImages = Array.from(new Set(
    sceneShots.map(sh => (sh as any).primaryCharacter?.portraitUrl).filter((u: any): u is string => !!u)
  )).slice(0, 9);

  for (const shot of sceneShots) {
    const { requestId, pollingUrl, mediaUrl } = await generateTextVideo({
      apiKey: segmindKey,
      model: "seedance",
      prompt: shot.videoPrompt || shot.soulPrompt || "",
      referenceImages,
      duration: (shot.duration as 5 | 10 | 15 | null) ?? 5,
    });

    if (mediaUrl) {
      await db.update(productionShots)
        .set({ finalVideoUrl: mediaUrl, generationStatus: "final_ready", updatedAt: new Date() })
        .where(eq(productionShots.id, shot.id));
    } else {
      await db.update(productionShots)
        .set({ generationStatus: "generating_final", higgsfieldJobId: `${requestId}|${pollingUrl}`, updatedAt: new Date() })
        .where(eq(productionShots.id, shot.id));
    }
  }

  return NextResponse.json({ status: "generating_final", shotCount: sceneShots.length });
}
```

export const dynamic = 'force-dynamic';
export const maxDuration = 300;

import { NextResponse } from "next/server";
import { getRequiredSession } from "@/lib/auth-helpers";
import { db } from "@/db";
import { projects, productionShots, users } from "@/db/schema";
import { eq, and } from "drizzle-orm";
import { pollJob } from "@/lib/higgsfield/client";
import { put } from "@vercel/blob";
import { decrypt } from "@/lib/crypto";
import { concatVideos } from "@/lib/video/concat";
import { writeFile, readFile, mkdtemp } from "node:fs/promises";
import { tmpdir } from "node:os";
import { join } from "node:path";

async function verifyOwnership(projectId: string, userId: string) {
  return db.query.projects.findFirst({
    where: and(eq(projects.id, projectId), eq(projects.userId, userId)),
  });
}

export async function GET(_: Request, { params }: { params: Promise<{ projectId: string; sceneNumber: string }> }) {
  const s = await getRequiredSession();
  const { projectId, sceneNumber: sceneNumberRaw } = await params;
  if (!await verifyOwnership(projectId, s.user.id))
    return NextResponse.json({ error: "Not found" }, { status: 404 });

  const sceneNumber = Number(sceneNumberRaw);
  let sceneShots = await db.query.productionShots.findMany({
    where: and(eq(productionShots.projectId, projectId), eq(productionShots.sceneNumber, sceneNumber)),
    orderBy: (sh, { asc }) => [asc(sh.shotNumber)],
  });
  if (sceneShots.length === 0) return NextResponse.json({ error: "Scene not found" }, { status: 404 });

  const alreadyStitched = sceneShots.find(sh => sh.sceneFinalVideoUrl);
  if (alreadyStitched) return NextResponse.json({ status: "final_ready", videoUrl: alreadyStitched.sceneFinalVideoUrl });

  const pending = sceneShots.filter(sh => sh.generationStatus === "generating_final" && sh.higgsfieldJobId);
  if (pending.length > 0) {
    const user = await db.query.users.findFirst({ where: eq(users.id, s.user.id) });
    const segmindKey = decrypt(user?.segmindApiKey ?? "");
    if (!segmindKey)
      return NextResponse.json({ error: "Add your Segmind API key in Settings." }, { status: 400 });

    for (const shot of pending) {
      const [, pollingUrl] = shot.higgsfieldJobId!.split("|");
      const { status, mediaUrl } = await pollJob({ apiKey: segmindKey, pollingUrl });

      if (status === "COMPLETED" && mediaUrl) {
        await db.update(productionShots)
          .set({ finalVideoUrl: mediaUrl, generationStatus: "final_ready", updatedAt: new Date() })
          .where(eq(productionShots.id, shot.id));
      } else if (status === "FAILED" || status === "ERROR") {
        await db.update(productionShots)
          .set({ generationStatus: "error", updatedAt: new Date() })
          .where(eq(productionShots.id, shot.id));
        return NextResponse.json({ status: "error" });
      }
    }

    // Re-read after this round of polling — some shots may have just completed.
    sceneShots = await db.query.productionShots.findMany({
      where: and(eq(productionShots.projectId, projectId), eq(productionShots.sceneNumber, sceneNumber)),
      orderBy: (sh, { asc }) => [asc(sh.shotNumber)],
    });
  }

  const stillPending = sceneShots.some(sh => sh.generationStatus !== "final_ready" || !sh.finalVideoUrl);
  if (stillPending) return NextResponse.json({ status: "generating_final" });

  // Every shot is final_ready — stitch them together in shot-number order.
  const workDir = await mkdtemp(join(tmpdir(), "scene-stitch-"));
  const localPaths: string[] = [];
  for (const shot of sceneShots) {
    const res = await fetch(shot.finalVideoUrl!);
    const buf = Buffer.from(await res.arrayBuffer());
    const localPath = join(workDir, `${shot.shotNumber}.mp4`);
    await writeFile(localPath, buf);
    localPaths.push(localPath);
  }

  const outputPath = join(workDir, "stitched.mp4");
  await concatVideos(localPaths, outputPath);
  const stitchedBuf = await readFile(outputPath);

  const blob = await put(
    `production/${projectId}/scene-${sceneNumber}/stitched-${Date.now()}.mp4`,
    stitchedBuf,
    { access: "public", contentType: "video/mp4" }
  );

  await db.update(productionShots)
    .set({ sceneFinalVideoUrl: blob.url, updatedAt: new Date() })
    .where(and(eq(productionShots.projectId, projectId), eq(productionShots.sceneNumber, sceneNumber)));

  return NextResponse.json({ status: "final_ready", videoUrl: blob.url });
}

## Self-Review Notes

- **Why no scene-level model restriction beyond Seedance carried over explicitly:** unchanged from the prior plan — this route is still only ever invoked with `model: "seedance"`, hardcoded, so the constraint holds structurally without needing a new check.
- **Why `multiShotScript`/Task 2-3 of the prior plan aren't reverted:** the column and prompt instructions are inert under the new code path (nothing reads them), and removing them is unrelated cleanup outside this plan's scope — flagged explicitly in Global Constraints rather than silently left for someone to wonder about later.
- **Concurrency note:** the status route's per-request "poll pending shots, then check if all are done, then stitch" sequence is not transactionally atomic — if two browser tabs poll the same scene simultaneously while the last shot completes, both could observe `stillPending = false` and both attempt to stitch. This duplicates work (wasted Blob storage + a second `sceneFinalVideoUrl` write) but does not corrupt data — the last write wins and the result is still a valid stitched video. Not fixed here: this matches the existing per-shot status route's pattern, which has the same class of benign double-poll race, and adding locking would be disproportionate to the actual risk for a feature reached by one user clicking one button.
