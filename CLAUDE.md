# GhostWriter AI
AI ghostwriting platform.

## Stack
Next.js 16, Drizzle ORM 0.45.x, Neon PostgreSQL, NextAuth, Anthropic Claude, Tailwind v4, Vercel.

## OpenAI policy
"No OpenAI for generation" applies to story/text generation only (Claude-only). OpenAI is permitted for: embeddings (`text-embedding-3-small`, craft library search) and Audio Novel TTS (`tts-1`, `src/app/api/audio/generate/route.ts`). Higgsfield replaces OpenAI for image/video.

## Known architectural debt ‚Äî read before adding more features
The app has two parallel UI shells (legacy `ToolbarPanel`-driven flow + the newer `WritingRoom` "One Path, Five Stages" shell), and complexity has been purely additive since the first commit (3 modes ‚Üí 26 modes; one flow ‚Üí two shells + a density dial to manage the resulting overload). Three orphaned-feature bugs found and fixed in the 2026-06-17 session (Add Chapter, Comic Studio, Audio Novel all unreachable from `WritingRoom`) were a direct symptom: the redesign meant to simplify the product is an *incomplete* port of the legacy shell, not a replacement for it. Full analysis, the first-commit comparison, and the recommended direction (finish one shell, retire the other, use progressive disclosure consistently instead of always-visible tool sprawl) are in `docs/product-history.md`. Don't add new top-level tools/panels without reading that file first.

## Architecture
- Continuity engine: chapter summaries in AI context
- Style DNA: reference works to 6 attributes
- Modes: Brainstorm / Outline / Write / 23 library modes + creator tools
- World Bible: characters (with contextVisibility), locations, plot threads
- Prompt caching: static/dynamic context split; static block cached (ephemeral)
- Context budget: buildStaticContext caps at 8,000 tokens via priority-ordered section assembly (`[Context trimmed ‚Äî project too large]` marker if exceeded); /api/ai/generate re-caps per subscription tier via capContextForTier (free/story_pro/creator_pro/all_access char limits; line-boundary-aware re-trim with `[Context truncated for tier limit]` marker if it re-trims)
- Model tiers: Haiku (free-tier generation + summaries/grading), Sonnet-4-6 (default generation), Opus-4-6 (quality/composition modes)
- Free tier: 10 gen/month on Haiku; 23 library modes gated (403 for free users)
- Voice fingerprinting: 10 stylometric markers extracted from last 5 chapters ‚Üí injected as binding constraints
- AIisms check: opt-in per project (Story Pro+) ‚Äî 20 fiction tells blocked post-generation
- Quality check: Tier 1 Haiku + Tier 2 Sonnet in parallel, non-blocking
- Toast system: src/lib/toast.ts + ToastContainer ‚Äî no external state manager
- Bundle: 10 heavy panels as dynamic imports (StoryHealth, Export, AltDraft, SprintMode, UpgradePrompt, CommandPalette, QualityReview, WorldBiblePanel, ToolbarPanel, StoryBible)
- Mode Registry (`src/lib/modes/registry.ts`): `MODE_REGISTRY` is the single source of truth for all 26 `GenerationMode`s ‚Äî label, modelTier, gate, qualityCheck, visibility, slash command, keywords, realismDomains. Derived by engine.ts (model tier + system prompts via `MI`), subscription.ts (`GATED_MODES`), formats.ts (`MODES`), useAIActions, ToolbarPanel
- Mode-aware context policy: each `MODE_REGISTRY` entry carries a `contextPolicy` (`needsCharacters`/`needsLocations`/`needsMemories`/`needsPlotThreads`/`needsRealism`/`charDepth: "full"|"brief"`) read by `buildStaticContext`/`buildDynamicContext`/`buildContext` (`src/lib/ai/context-builder.ts`) to gate which context sections are assembled per mode (e.g. horror/combat/action drop memories+plotThreads in favor of realism; brainstorm/outline/atmosphere use brief character lines). No `mode` (quick-start, alt-draft) falls back to "include everything"
- `useAIActions` (`src/hooks/useAIActions.ts`, ~67 lines) composes 5 focused hooks in `src/hooks/`: `ai-shared.ts` (shared `callAI`/`buildNeighbourContext`), `useGeneration` (core generate + undo + quality check), `useEntitySync` (auto-extraction suggestions), `usePipelines` (multi-agent pipeline mode), `useProseTools` (expand/tighten/show-dont-tell/subtext/rewrite)
- System prompts centralized in `src/lib/ai/prompts.ts` (30 exports): `SCREAMING_SNAKE_CASE` consts ending `_SYSTEM_PROMPT` for static prompts, camelCase `...SystemPrompt(...)` functions for interpolated ones ‚Äî covers creator-tool routes, knowledge-audit, tension-curve, transportation-check, villain-pov, alt-draft, production/generate-package, pipeline, and prose sub-operations (separate from the mode `MI` dispatch map)
- Higgsfield video models centralized in `src/lib/higgsfield/models.ts`: `VIDEO_MODELS` is the single source of truth (`VIDEO_ENDPOINTS`/`VIDEO_MODEL_INFO`/`ACTIVE_VIDEO_MODELS`/`MODE_TO_MODEL` all derived from it); `sora` marked `deprecated: true` (excluded from active/auto-selection), `MODE_TO_MODEL.horror` ‚Üí `veo`
- "One Path, Five Stages" UI redesign (`writingRoomShell`/`homeRedesign` GrowthBook flags ‚Äî `writingRoomShell` **ON** in production since 2026-06-15, `homeRedesign` still OFF): `WritingRoom.tsx` replaces the toolbar-driven flow with an Idea‚ÜíStructure‚ÜíDraft‚ÜíPolish‚ÜíExport stage ladder (`src/lib/guide/next-action.ts` `currentStage()`); `Home.tsx` replaces the dashboard (pending `homeRedesign`); `/` slash menu routes into the existing `ToolbarPanel` via an Actions overlay; creator formats get remapped stage labels (Angle/Outline-Hooks/Script/Retention edit/Publish pack) with per-stage tool rows
- Guide Engine: `nextAction(project)` (`src/lib/guide/next-action.ts`) returns one actionable suggestion from a chapter-level ladder; surfaced via `GuideBar` + `/api/events` (`guide_clicked`/`guide_dismissed`); dismissal state in `projects.dismissedGuideIds` jsonb
- Story Bible overlay (`src/components/StoryBible.tsx`, writingRoomShell-gated): full-screen Cast/World/Threads CRUD; auto-extraction suggestion chip (`EntitySuggestionsChip` + `src/lib/ai/entity-extraction.ts`) proposes Story Bible field updates from generated write-mode text
- Beat detection: `classifyBeat()` (`src/lib/modes/classify.ts`) matches a drafted beat against the 23 library modes' keywords; surfaced as `BeatDetectionChip` in WritingRoom's Draft stage
- Series/Universe architecture: storyType (linear/series/universe-story/parallel) per project; universes table with characters + events; series context chains previous books' memories; universe context injects canonical events + character states
- Voice fingerprint display: visible in WorldBiblePanel settings when 3+ chapters exist
- Cost monitoring: GET /api/admin/cost-report (ADMIN_SECRET-gated) ‚Äî 30-day blended cost estimate by model, top users, top modes
- Subscriptions: every account gets a `subscriptions` row at creation (credentials register + OAuth `events.createUser`, both upsert-safe); `scripts/backfill-subscriptions.js` seeds pre-existing accounts
- Subscription tier is verified server-side from the Razorpay subscription's `notes.tier`/`notes.userId` (set at creation in `POST /api/subscription`) in both `/api/subscription/verify` and `/api/webhooks/razorpay` ‚Äî client-supplied tier is never trusted
- 7-day Story Pro trial: `users.trialEndAt` set at registration, checked in `getUserTier()` ‚Äî free users with an unexpired trial get story_pro access until it lapses
- Email verification: token-based (`email_verification_tokens` table), sent at registration; `/api/auth/verify-email?token=...` sets `users.emailVerified`; settings shows a resend banner if unverified
- Rate limiting: `checkAiRateLimit` fails CLOSED (503) in production if Upstash is unconfigured or throws; dev fails open
- Stripe fully removed (portal/webhook routes, STRIPE_PRICES, npm deps, copy) ‚Äî Razorpay Subscriptions only; dead `stripeCustomerId`/`stripeSubscriptionId` columns dropped from `subscriptions` table (migration `0006_fantastic_luminals.sql`, pushed live 2026-06-15)
- Razorpay test-mode quirk: `subscriptions.create`/`cancel` in `/api/subscription` wrapped in `withAuthRetry()` (4 attempts, backoff) ‚Äî Razorpay's test API intermittently 401s with valid credentials
- Content persistence: `updateChapter(field, value)` in `useProjectState` accumulates fields in `pendingChangesRef` and fires a single debounced PATCH (1500ms) ‚Äî prevents content loss from rapid multi-field updates; chapter/project IDs stored inside ref to prevent stale-closure writes to wrong chapter
- Editor source-of-truth: Write-mode generation inserts via `ChapterEditorHandle.insertContent()` (TipTap forwardRef); non-write modes also thread through the same ref via `insertIntoEditorRef` in GhostWriterApp; `forceEditor` state bypasses stage gating with "Write" button in stage header
- Format capability map: `FORMAT_CONFIG` in `src/lib/formats.ts` ‚Äî `FormatCapabilities` interface + `formatHasCohost()`/`formatUsesSegments()` helpers replace scattered `format === "Podcast Episode"` magic-string checks
- Density dial: `useDensity()` hook (localStorage `gw:density`) + `minDensity` field on ModeConfig ‚Äî ToolbarPanel shows Simple/Standard/Full pill; Simple shows 8 core modes, Standard adds 13 specialized, Full adds 4 advanced
- AI Initiative: per-project `aiInitiative` column (Leads/Collaborates/Assists, default Collaborates) ‚Äî Leads auto-fires `generate()` 4s after guide suggestion; Assists hides GuideBar; set in WorldBiblePanel Settings tab
- Reader sessions: full share flow now reachable ‚Äî "‚Üó Share Draft" button in WritingRoom stage header creates token + copies `/reader/[token]` URL to clipboard; reactions/heatmap fully wired
- Sprint Mode: word-count goal (250/500/1000/custom presets), progress bar, goal-reached celebration banner; goal locks once writing begins
- Fix Weakness: `/api/ai/prose-fix` accepts `{ text, fixInstruction }` ‚Äî StoryHealthPanel weakness items now show "Fix This" button; result shown as inline diff with "Apply to Chapter"
- Audio Novel + lipsync: extracted into shared `AudioNovelPanel` (`src/components/AudioNovelPanel.tsx`) ‚Äî generate button, audio player, lipsync character-portrait selection, calls `/api/audio/generate` + `/api/audio/lipsync` with manual status polling. Used by the legacy `panels/ChapterEditor.tsx` sidebar AND by `WritingRoom`'s footer (toggleable "üéß Audio Novel" button, Draft/Polish stages) ‚Äî previously only reachable via the legacy sidebar, which `WritingRoom` never renders, making it unreachable whenever `writingRoomShell` is ON (fixed 2026-06-17)
- Surgical Edit: `/api/ai/surgical-edit` accepts `{ chapterContent, instruction }` ‚Üí Claude locates passage + returns `{ found, replacement, explanation, updatedJson }` ‚Üí WritingRoom "‚úè Find & Edit" panel shows red/green diff, "Apply Change" calls `editorRef.current?.replaceContent()`
- CraftDepthChip: shows when `activeInfluence` WorkPacket is set in Draft stage ‚Äî displays influence title + top craft principle for the active mode; dismiss clears `activeInfluence`
- Chapter creation in WritingRoom: "+ Add {chapter label}" button in the header (next to `‚Äπ ‚Ä∫` chapter nav) calls `useProjectState`'s `addChapter()` ‚Äî previously WritingRoom had no way to create a chapter at all, so all AI output accumulated in chapter 1 regardless of outline/beat count (fixed 2026-06-17)
- Stage navigation in WritingRoom: the Idea/Structure/Draft/Polish/Export progress pills are clickable (`manualStage` state overrides the auto-computed `currentStage()`) ‚Äî previously the displayed stage was 100% auto-computed with no way back once the algorithm advanced past a stage (e.g. once any single chapter passed the word-count threshold, the view jumped to Export with no way to return to Draft for other chapters) (fixed 2026-06-17)
- Comic Studio reachability: `WritingRoom`'s Export stage has a "üé® Open Comic Studio ‚Üí" button (gated by `isStoryFormat`) that reuses the existing Actions-drawer mechanism already used for Production Studio (`setShowComicStudio(true); setActionsOpen(true)`) ‚Äî previously `ComicStudio.tsx` was only rendered inside the legacy `ToolbarPanel`, making it unreachable whenever `writingRoomShell` is ON (fixed 2026-06-17)
- Series Bible ‚Üí AI generation: `buildSeriesBibleContext()` in `/api/ai/generate/route.ts` looks up whether the current project is linked to a `series_bibles` row (via `projectIds` array containment) and injects premise/tone/worldRules/seriesCharacterArcs/continuityNotes/timeline into the static (cached) context block, alongside the existing `buildSeriesUniverseContext()`. Previously the Series Bible CRUD UI/API was fully built but had zero effect on generation ‚Äî editing it did nothing (fixed 2026-06-17)
- Editor text visibility/layout: `.chapter-editor .ProseMirror` (`src/app/globals.css`) used a viewport-relative padding formula (`(100vw - 680px) / 2`) assuming the editor spans the full browser width; in `WritingRoom`/`WritePanel` it shares the row with sidebars (~490px), so real content width collapsed to a constant ~190px regardless of screen size, wrapping nearly every word onto its own line. Also used the dark-theme default text color (`--color-text-primary`, near-white) against the redesign's light app shell, making text invisible. Fixed via container-relative `max-width:680px; margin:0 auto` (matches the working `.prose-content` pattern already in the file) + a `lightTheme` prop on `ChapterEditor` (`chapter-editor--light` class) used by `WritingRoom`/`WritePanel` only ‚Äî `SprintMode`'s genuinely full-viewport dark editor is untouched (fixed 2026-06-17)
- Adapt (cross-format conversion, v1 Novel‚ÜíScreenplay only): `src/components/AdaptPanel.tsx`, opened from a "üé≠ Adapt this story ‚Üí" button in `ExportStageView` (gated `isStoryFormat`). `POST /api/projects/[id]/adapt` creates a linked target project (`projects.adaptedFromProjectId`) and copies World Bible rows (characters/locations/plotThreads ‚Äî fresh ids, cross-reference link arrays reset since they'd point at the wrong project). Client then loops `POST /api/projects/[newId]/adapt-chapter` once per source chapter (sequential, never parallel, to keep `sortOrder` deterministic) ‚Äî each call is a dedicated one-off Claude call (not the `MODE_REGISTRY`/`MI` dispatch, since this transforms existing content rather than generating new content from a prompt) using `getFormatRules("Screenplay")` from `engine.ts`, metered via `meterAndGate(userId, "adapt-chapter")` (1 credit/chapter, same weight as `generate`). Partial failure (e.g. quota hit mid-conversion) keeps already-converted chapters ‚Äî the new project is never rolled back. Full design: `docs/superpowers/specs/2026-06-18-adapt-cross-format-design.md`. Other conversion targets (Web Series, Comic, Higgsfield, Novelization) are visible in the menu as disabled "Coming soon" ‚Äî only the capability map needs extending when they ship, not the UI.
- Prose-augmentation helpers, split 2026-06-21 per a six-model blind panel (see item 23): `buildPromiseLedger` (`src/lib/ai/promise-ledger.ts`, DB-only unresolved-promise aggregation) and `buildVoiceExemplars` (`src/lib/ai/exemplars.ts`, top-2 embedding-similarity craft-library retrieval) run **unconditionally** (no flag) for any qualifying request ‚Äî `isProseMode(mode) && tier !== 'free' && projectId`, checked before anything else so non-qualifying requests pay zero added latency. The costly Haiku scene-blueprint (`buildSceneBlueprint`, `src/lib/ai/scene-blueprint.ts`) is the ONLY piece still behind a flag ‚Äî `sceneBlueprint` (default OFF), checked server-side via `isFeatureOnServer` (`src/lib/growthbook-server.ts`, the only server-side GrowthBook check in the codebase) ‚Äî and additionally respects the per-request `skipBlueprint` escape hatch. All three remain individually fail-open (return `""` on error) and land in the dynamic (never cached) context block. `src/lib/analysis/rhythm.ts` (deterministic, no LLM) surfaces a read-only signal in `PolishStageView`, unconditionally (no flag ‚Äî same free-subset reasoning). `/api/ai/refine` (critic-editor polish pass, `refinePassage()` in `engine.ts`) is metered at the cheaper `"refine"` weight (0.5 credits), not full `"generate"`.
- Streaming (`streaming` flag, client-only, default OFF): `generateStream()` (`engine.ts`) swaps `messages.create` for `messages.stream` with an `onDelta` callback, otherwise identical context assembly to `generate()`. `/api/ai/generate`'s `stream: true` branch returns a `ReadableStream` (`text/plain`, not JSON) instead of the usual JSON body, reusing the exact same `finalDynamic` (so streamed generations get the same quality_stack augmentation as non-streamed). Client side: `callAIStream()` (`src/hooks/ai-shared.ts`) detects a JSON content-type to fall back cleanly when the server returns a control response (`upgrade_required`, violation confirmation, rate-limit) instead of actually opening a stream. `WritingRoom.tsx`'s `runGenerate()` passes `editorStream` (start/delta/end callbacks) only when the flag is on; `ChapterEditorHandle.streamStart/streamDelta/streamEnd` (`ChapterEditor.tsx`) lock the editor and insert raw deltas for a live typewriter effect, then reparse into clean TipTap nodes and save once on completion ‚Äî no separate "save-debounce at stream end" logic needed, it falls out of the existing 1500ms `onUpdate` debounce for free.

## Auth
Application-level ownership checks only. Never use Supabase RLS. Always call getRequiredSession() in protected routes. For child-resource routes (a shot, panel, chapter looked up by its own ID), use `verifyChildOwnership(table, childId, projectId)` (`src/lib/auth-helpers.ts`) to confirm it belongs to the caller's project.

## LSP false positives
"Props must be serializable" warnings for function props between 'use client' components are pre-existing false positives. tsc --noEmit exit 0 is ground truth.
