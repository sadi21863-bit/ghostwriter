# GhostWriter AI
AI ghostwriting platform.

## Stack
Next.js 16, Drizzle ORM 0.45.x, Neon PostgreSQL, NextAuth, Anthropic Claude, Tailwind v4, Vercel.

## OpenAI policy
"No OpenAI for generation" applies to story/text generation only (Claude-only). OpenAI is permitted for: embeddings (`text-embedding-3-small`, craft library search) and Audio Novel TTS (`tts-1`, one of two user-selectable providers as of 2026-07-07 — see item 51). Higgsfield replaces OpenAI for image/video.

## Known architectural debt — read before adding more features
**The dual-shell debt is RESOLVED as of 2026-06-29** (commit `119fce2`, plan `docs/superpowers/plans/2026-06-29-kill-dual-shell.md`). There is now ONE shell: `WritingRoom` (the "One Path, Five Stages" Idea→Structure→Draft→Polish→Export flow). The legacy `ToolbarPanel`-driven always-visible layout + the legacy `panels/ChapterEditor.tsx` + the `writingRoomShell` GrowthBook flag were hard-deleted. **`ToolbarPanel` itself still exists** — it's reused as the on-demand Actions-drawer content inside WritingRoom (`GhostWriterApp.tsx`), NOT as a second shell. The modern editor is `@/components/editor/ChapterEditor` (used by WritingRoom/SprintMode/WritePanel). `src/components/__tests__/live-shell-reachability.test.ts` is now a regression guard: it fails if the legacy shell/flag is resurrected OR if a must-be-reachable feature stops being imported by the live shell. Historical context (the first-commit comparison, why the additive complexity happened, the 2026-06-17 orphaned-feature fixes that preceded this) is in `docs/product-history.md`. Remaining additive-complexity debt (3 modes → 26 modes) is unchanged — don't add new top-level tools/panels without reading `docs/product-history.md` first. (Density dial is no longer part of this debt as of 2026-07-06 — it's DB-persisted per-project, not an ungoverned localStorage sprawl point.) Next planned consolidation steps (not yet done): zod-typed JSONB story blobs, a `MODE_REGISTRY` core/experimental split, and the Discover/Shape/Write/Produce + Writer/Director/Editor restructure.

## Architecture
- Continuity engine: chapter summaries in AI context
- Style DNA: reference works to 6 attributes
- Modes: Brainstorm / Outline / Write / 23 library modes + creator tools
- World Bible: characters (with contextVisibility), locations, plot threads, **world entities** (objects/weapons/organizations/factions/phenomena/entities/concepts — one kind-tagged `world_entities` table, migration 0015; zod-typed `properties` JSONB via `decodeWorldEntityProperties`/`WorldEntityKindSchema` in `src/lib/types/story.ts`; CRUD at `/api/projects/[id]/world-entities`; grouped into 3 sections in the Story Bible "Elements" tab; surfaced as `world_entity` nodes + `involves` edges in the Story Graph; injected into AI **prose** context as a `WORLD ELEMENTS` section gated by `ContextPolicy.needsWorldEntities` (opt-in: combat/horror/action + no-mode), AND into **comic** panel/breakdown prompts (`buildPanelPrompt`/`buildBreakdownPrompt`, relevance = alwaysInContext or named-in-action) and **video** production-package prompts; auto-extraction suggestion chip proposes summary/description updates via the `worldEntities` EntityKey. Spec/plan: `docs/superpowers/{specs,plans}/2026-06-30-world-entities*.md`)
- Prompt caching: static/dynamic context split; static block cached (ephemeral)
- Context budget: buildStaticContext caps at 8,000 tokens via priority-ordered section assembly (`[Context trimmed — project too large]` marker if exceeded); /api/ai/generate re-caps per subscription tier via capContextForTier (free/story_pro/creator_pro/all_access char limits; line-boundary-aware re-trim with `[Context truncated for tier limit]` marker if it re-trims)
- Model tiers: Haiku (free-tier generation + summaries/grading), Sonnet 5 (default generation, `claude-sonnet-5` — switched from Sonnet 4.6 2026-07-01), Opus-4-6 (quality/composition modes)
- Free tier: 10 gen/month on Haiku; 23 library modes gated (403 for free users)
- Voice fingerprinting: 10 stylometric markers extracted from last 5 chapters → injected as binding constraints
- AIisms check: opt-in per project (Story Pro+) — 20 fiction tells blocked post-generation
- Quality check: Tier 1 Haiku + Tier 2 Sonnet in parallel, non-blocking
- Toast system: src/lib/toast.ts + ToastContainer — no external state manager
- Bundle: 10 heavy panels as dynamic imports (StoryHealth, Export, AltDraft, SprintMode, UpgradePrompt, CommandPalette, QualityReview, WorldBiblePanel, ToolbarPanel, StoryBible)
- Mode Registry (`src/lib/modes/registry.ts`): `MODE_REGISTRY` is the single source of truth for all 26 `GenerationMode`s — label, modelTier, gate, qualityCheck, visibility, slash command, keywords, realismDomains. Derived by engine.ts (model tier + system prompts via `MI`), subscription.ts (`GATED_MODES`), formats.ts (`MODES`), useAIActions, ToolbarPanel
- Mode-aware context policy: each `MODE_REGISTRY` entry carries a `contextPolicy` (`needsCharacters`/`needsLocations`/`needsMemories`/`needsPlotThreads`/`needsRealism`/`charDepth: "full"|"brief"`) read by `buildStaticContext`/`buildDynamicContext`/`buildContext` (`src/lib/ai/context-builder.ts`) to gate which context sections are assembled per mode (e.g. horror/combat/action drop memories+plotThreads in favor of realism; brainstorm/outline/atmosphere use brief character lines). No `mode` (quick-start, alt-draft) falls back to "include everything"
- `useAIActions` (`src/hooks/useAIActions.ts`, ~67 lines) composes 5 focused hooks in `src/hooks/`: `ai-shared.ts` (shared `callAI`/`buildNeighbourContext`), `useGeneration` (core generate + undo + quality check), `useEntitySync` (auto-extraction suggestions), `usePipelines` (multi-agent pipeline mode), `useProseTools` (expand/tighten/show-dont-tell/subtext/rewrite)
- System prompts centralized in `src/lib/ai/prompts.ts` (30 exports): `SCREAMING_SNAKE_CASE` consts ending `_SYSTEM_PROMPT` for static prompts, camelCase `...SystemPrompt(...)` functions for interpolated ones — covers creator-tool routes, knowledge-audit, tension-curve, transportation-check, villain-pov, alt-draft, production/generate-package, pipeline, and prose sub-operations (separate from the mode `MI` dispatch map)
- Higgsfield video models centralized in `src/lib/higgsfield/models.ts`: `VIDEO_MODELS` is the single source of truth (`VIDEO_ENDPOINTS`/`VIDEO_MODEL_INFO`/`ACTIVE_VIDEO_MODELS`/`MODE_TO_MODEL` all derived from it); `sora` marked `deprecated: true` (excluded from active/auto-selection), `MODE_TO_MODEL.horror` → `veo`
- "One Path, Five Stages" UI redesign: `WritingRoom.tsx` is the app's only shell (see architectural-debt note above — the `writingRoomShell` flag that used to gate this was removed 2026-06-29 once it was confirmed feature-complete) — an Idea→Structure→Draft→Polish→Export stage ladder (`src/lib/guide/next-action.ts` `currentStage()`). `Home.tsx` replaces the dashboard (still pending, behind the still-live `homeRedesign` flag, currently OFF). `/` slash menu routes into `ToolbarPanel` (reused as Actions-drawer content) via an Actions overlay; creator formats get remapped stage labels (Angle/Outline-Hooks/Script/Retention edit/Publish pack) with per-stage tool rows

## Auth
Application-level ownership checks only. Never use Supabase RLS. Always call getRequiredSession() in protected routes. For child-resource routes (a shot, panel, chapter looked up by its own ID), use `verifyChildOwnership(table, childId, projectId)` (`src/lib/auth-helpers.ts`) to confirm it belongs to the caller's project.

## LSP false positives
"Props must be serializable" warnings for function props between 'use client' components are pre-existing false positives. tsc --noEmit exit 0 is ground truth.
