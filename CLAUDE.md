# GhostWriter AI
AI ghostwriting platform.

## Stack
Next.js 16, Drizzle ORM 0.45.x, Neon PostgreSQL, NextAuth, Anthropic Claude, Tailwind v4, Vercel.

## OpenAI policy
"No OpenAI for generation" applies to story/text generation only (Claude-only). OpenAI is permitted for: embeddings (`text-embedding-3-small`, craft library search) and Audio Novel TTS (`tts-1`, one of two user-selectable providers as of 2026-07-07 ŌĆö see item 51). Higgsfield replaces OpenAI for image/video.

## Known architectural debt ŌĆö read before adding more features
**The dual-shell debt is RESOLVED as of 2026-06-29** (commit `119fce2`, plan `docs/superpowers/plans/2026-06-29-kill-dual-shell.md`). There is now ONE shell: `WritingRoom` (the "One Path, Five Stages" IdeaŌåÆStructureŌåÆDraftŌåÆPolishŌåÆExport flow). The legacy `ToolbarPanel`-driven always-visible layout + the legacy `panels/ChapterEditor.tsx` + the `writingRoomShell` GrowthBook flag were hard-deleted. **`ToolbarPanel` itself still exists** ŌĆö it's reused as the on-demand Actions-drawer content inside WritingRoom (`GhostWriterApp.tsx`), NOT as a second shell. The modern editor is `@/components/editor/ChapterEditor` (used by WritingRoom/SprintMode/WritePanel). `src/components/__tests__/live-shell-reachability.test.ts` is now a regression guard: it fails if the legacy shell/flag is resurrected OR if a must-be-reachable feature stops being imported by the live shell. Historical context (the first-commit comparison, why the additive complexity happened, the 2026-06-17 orphaned-feature fixes that preceded this) is in `docs/product-history.md`. Remaining additive-complexity debt (3 modes ŌåÆ 26 modes) is unchanged ŌĆö don't add new top-level tools/panels without reading `docs/product-history.md` first. (Density dial is no longer part of this debt as of 2026-07-06 ŌĆö it's DB-persisted per-project, not an ungoverned localStorage sprawl point.) Next planned consolidation steps (not yet done): zod-typed JSONB story blobs, a `MODE_REGISTRY` core/experimental split, and the Discover/Shape/Write/Produce + Writer/Director/Editor restructure.

## Auth
Application-level ownership checks only. Never use Supabase RLS. Always call getRequiredSession() in protected routes. For child-resource routes (a shot, panel, chapter looked up by its own ID), use `verifyChildOwnership(table, childId, projectId)` (`src/lib/auth-helpers.ts`) to confirm it belongs to the caller's project.
